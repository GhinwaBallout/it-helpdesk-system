# IT Help Desk & Ticketing Management System

## Project Overview

The IT Help Desk & Ticketing Management System is a web app that helps organizations manage IT support requests efficiently. It provides one central platform where employees report issues, managers assign and monitor requests, IT support agents resolve them, and administrators oversee the system.

Instead of emails, calls, or verbal requests, every issue is logged as a ticket. Each ticket moves through a clear workflow — creation, assignment, resolution, and closure — so everyone can track its status in real time.

## Problem Statement

Many organizations still handle IT support through emails, phone calls, or informal communication. This makes it hard to track requests, assign responsibility, monitor progress, and keep historical records. Without a centralized system, requests can get delayed, duplicated, or lost — hurting employee productivity and making it difficult for managers to evaluate IT team performance. The proposed system solves this by offering a structured platform to submit, manage, track, and resolve IT tickets, improving communication between employees and the IT department.

## System Users & Roles

- **Admin**: Full system access. Manage users, departments, ticket categories, reports, and all system settings.
- **IT Support Agent**: Manage assigned tickets, update ticket status, communicate with employees, resolve and close tickets.
- **Employee**: Create support tickets, upload attachments, track ticket status, add comments, and view ticket history.
- **Manager**: Monitor all tickets, assign tickets to IT Support Agents, review reports, and oversee support operations.

## Project Design & Documentation

### Workflow Diagrams
Workflow diagrams (ticket lifecycle: creation → assignment → resolution → closure, and role-based process flows) are designed using **draw.io**.

### UI Wireframes
UI wireframes for the platform (employee dashboard, agent ticket view, manager overview, admin panel) are designed using **Replit**.

### Database Schema & ERD Diagram
The database schema and Entity-Relationship Diagram (ERD), including tables for Users, Tickets, Departments, Categories, Comments, and Attachments, along with their relationships, are created using **dbdiagram.io**.
